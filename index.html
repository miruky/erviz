SQL DDL → ER 図

CREATE TABLE から、
関係を描き出す。

マイグレーションや pg_dump のDDLをそのまま貼り付けると、テーブルと外部キーを読み取って ER図を組み立てる。専用記法への書き直しはいらない。図はSVGとして手元で生成され、スキーマがサーバーへ送られることはない。

      仕組み

      DDLからSVGまで

      読めない文で止まらない

      方言の海であるSQLを完全に解釈する道は選ばない。読めた分だけ描いて、読めなかった文は画面の警告に残す。図を出すのが目的のツールでは、エラーで何も出ないのが一番役に立たない。

      レイアウトは決定的に

      外部キーの依存から層番号を決め、同じDDLからは常に同じ図を作る。乱数も物理シミュレーションも使わないので、図をGit管理しても差分はスキーマの変更だけを映す。

      SVGは単体で配れる形に

      出力にはviewBoxとスタイルが埋め込まれ、特定のページに依存しない。リポジトリにコミットすればそのままプレビューでき、閲覧側のライト・ダークにも追従する。
